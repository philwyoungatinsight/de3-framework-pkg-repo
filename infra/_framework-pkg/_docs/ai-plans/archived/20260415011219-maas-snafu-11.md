## The `mgmt_wake_via_plug: true` Rule (ms01-01)

ms01-01's AMT loses standby power when the machine is fully off. Bouncing the smart
plug restores AC power → AMT wakes up → wsman can set the PXE boot override normally.

**The plug's only job is to get AMT working. AMT's job is to tell the machine what to boot.**

Without AMT, a raw plug cycle boots whatever BIOS says (possibly GRUB from disk).
With AMT, wsman sets the one-time PXE boot override before power-cycling.

This is why `mgmt_wake_via_plug: true` triggers a bounce-then-retry sequence, NOT
a raw plug cycle.

## What Broke (maas-snafu-10 → maas-snafu-11)

### Original snafu-10 issue
"Restore AMT standby" only ran for machines in `['Commissioning', 'Testing', 'Deploying']`
states. At `commissioning:pre`, machines are in `New` or `Ready` — the restore was
skipped, AMT never woke up, MaaS failed its power commands.

**Fix (commit `68fb5175`)**: extended the `when` condition to also run at
`commissioning:pre` and `deploying:pre` regardless of MaaS status.

### Discovered during this session
`maas-machines-precheck` pre-emptively bounced the plug before trying wsman, even when
AMT was already reachable. The bounce → wait was the first step, not a fallback.

Also, ms01-02 and ms01-03 had no `smart_plug_host` in config, so the fallback was silently
skipped when their wsman failed.

**Why the last run got stuck**: ms01-02 wsman rc=1, no smart_plug_host → no fallback →
enrollment poll waited for all 4 machines → timed out.

---

## Fixes Applied (this session)

### 1. Added smart_plug_host to ms01-02 and ms01-03

```yaml
# ms01-02
smart_plug_host: 192.168.2.105
smart_plug_type: tapo

# ms01-03
smart_plug_host: 192.168.2.210
smart_plug_type: tapo
```

### 2. Rewrote AMT power cycle sequence in `maas-machines-precheck`

Old (wrong) sequence:
1. Pre-emptively bounce plug if AMT unreachable (mgmt_wake_via_plug only)
2. Run wsman
3. If wsman fails and has smart_plug_host → raw plug fallback

New (correct) sequence:
1. **Try wsman first** (attempt 1) for all AMT machines
2. If attempt 1 failed AND `mgmt_wake_via_plug: true`:
   - Bounce smart plug → wait up to 120s for AMT port 16993
   - **Retry wsman** (attempt 2) — AMT sets PXE boot override, better than raw plug
3. Last resort raw plug cycle for machines where all wsman attempts failed AND
   `smart_plug_host` is configured (PXE boots via BIOS order, which must be PXE-first
   per one-time setup pre-condition)
