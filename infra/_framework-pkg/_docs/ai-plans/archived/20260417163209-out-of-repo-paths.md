# Plan: Eliminate remaining out-of-repo runtime paths

## Objective

Remove the three remaining paths that are created or read outside the git repo,
making the codebase fully self-contained and safe to run from two checkouts
simultaneously without interference.

## Context

Three real hits remain after the previous `dynamic-dir-runtime-paths` cleanup:

| # | File | Path | Problem |
|---|------|------|---------|
| 1 | `utilities/python/validate-config.py:153` | `~/.cache/pwy-home-lab/validate-config-last-run` | Rate-limit flag for config validation; shared across all repo checkouts |
| 2 | `infra/de3-gui-pkg/.../homelab_gui.py:50` | `~/git/de3` | Hardcoded fallback for `_STACK_DIR`; user-specific, breaks for anyone else |
| 3 | `infra/de3-gui-pkg/.../homelab_gui.py:10024` | `/tmp/gui_preview.html` | HTML redirect for PyCharm preview; shared filename → two instances overwrite |

### Fix approach

**1. `validate-config.py` flag file**  
The script already computes `root = git_root()`. Change flag path from
`Path.home() / flag_file` to `root / "config" / "tmp" / "validate-config-last-run"` —
derived directly from the repo root, no env var needed, no yaml key needed.
Remove `flag_file` from `config/framework.yaml`.

**2. `_STACK_DIR` default in `homelab_gui.py`**  
The env var is always set by `set_env.sh`; the fallback only fires if someone
imports the module without sourcing `set_env.sh`. Replace the hardcoded
`Path.home() / "git/de3"` with a function that walks up from `__file__` looking
for `set_env.sh` as the repo-root sentinel:

```python
def _find_repo_root(start: Path) -> Path:
    for p in [start, *start.parents]:
        if (p / "set_env.sh").exists():
            return p
    return start  # last resort — caller will get an error eventually

_STACK_DIR = Path(os.environ.get("_STACK_DIR") or str(_find_repo_root(Path(__file__).resolve())))
```

**3. `/tmp/gui_preview.html`**  
Replace the hardcoded string with `Path(os.environ.get("_GUI_DIR", "/tmp")) / "gui_preview.html"`.
