# Plan: Fix `./run -A de3-gui`

## Objective

`./run -A de3-gui` fails with `ERROR: waves_ordering.yaml not found in config/ or any
infra/*/_config/` before even reaching the app runner. This plan fixes two bugs in the
`run` script: the broken `waves_ordering.yaml` discovery (which affects ALL modes, not
just `-A`) and the unnecessary config load that blocks `-A` from running.

## Context

**Bug 1 — `waves_ordering.yaml` search doesn't check `_framework_settings/`**

`load_all_configs()` (run:276–287) searches for `waves_ordering.yaml` in:
- `config/waves_ordering.yaml`
- `infra/*/_config/waves_ordering.yaml`

The actual file lives at:
`infra/pwy-home-lab-pkg/_config/_framework_settings/waves_ordering.yaml`
(and also `infra/_framework-pkg/_config/_framework_settings/waves_ordering.yaml` as
the framework default).

The `_framework_settings/` subdirectory convention is already handled correctly by
`framework_config.py`'s `find_framework_config_dirs()`, but `load_all_configs()` has its
own hand-rolled search that predates this convention and was never updated.

This means **every `./run` invocation is currently broken** — not just `-A`.

**Bug 2 — `-A` invokes `load_all_configs()` unnecessarily**

`main()` calls `cfg = load_all_configs()` unconditionally at line 871. The `-A` handler
is only reached at line 928. Since `-A` just runs `make` in an `_application/` directory,
it needs no wave config at all. The fix is to handle `-A` before `load_all_configs()`.

**`find_framework_config_dirs()` search order** (lowest → highest priority):
1. `infra/_framework-pkg/_config/_framework_settings/`
2. `$_FRAMEWORK_CONFIG_PKG_DIR/_config/_framework_settings/`  (e.g. `infra/pwy-home-lab-pkg/…`)
3. `config/`

`waves_ordering.yaml` in the highest-priority dir that has it wins.

## Files to Create / Modify

### `run` — modify

**Change 1: Fix `waves_ordering.yaml` discovery (lines 276–287)**

Replace the hand-rolled search with one that mirrors `find_framework_config_dirs()`:

```python
    # Discover waves_ordering.yaml using the same search dirs as framework_config.py.
    # Later dirs have higher priority; iterate in reverse so the first match is the winner.
    ordering_file = None
    for config_dir in reversed(find_framework_config_dirs(STACK_ROOT)):
        candidate = config_dir / 'waves_ordering.yaml'
        if candidate.exists():
            ordering_file = candidate
            break
    if ordering_file is None:
        sys.exit('ERROR: waves_ordering.yaml not found in any framework config dir')
```

This replaces the entire block from the comment `# Discover waves_ordering.yaml...`
through `if ordering_file is None: sys.exit(...)`.

**Change 2: Move `-A` handling before `load_all_configs()` (line 871)**

After `args = parse_args()` and before `cfg = load_all_configs()`, insert:

```python
    # ── app (no logging, run make in _application/<name>) ──
    # Handled here — before load_all_configs() — because apps need no wave config.
    if args.app is not None:
        app_dir = find_application(args.app)
        print(f'Running make in {app_dir}')
        result = subprocess.run(['make'], cwd=str(app_dir), env=ENV)
        sys.exit(result.returncode)
```

Then **delete** the duplicate block at line 927–932:
