# Plan: maas-snafu-10 — Comprehensive Pre/Post Gate Checks for Every MaaS Lifecycle Wave

## Objective

Add strict, zero-trust pre AND post gate checks to every `maas.lifecycle.*` wave.
Every check re-reads state from scratch — nothing is assumed from prior waves.
A machine that passes a gate is actually in the expected state; a machine that fails a gate
stops the wave immediately with a clear error message. No optimism. No silent skips.

---

## Context

### What broke today (the incident that prompted this plan)

1. `maas-lifecycle-sanity` (pre-check for commissioning) correctly detected power_type mismatch
   and annihilated 4 machines (deleted from MaaS + wiped GCS state). ✓
2. The sanity check then exited **success**. The commissioning apply immediately ran.
3. Machine units had no TF state → mock outputs → `system_id = "placeholder-system-id"`.
4. `trigger-commission.sh` has a guard: placeholder → skip commissioning. So it skipped silently.
5. Commission unit wrote `system_id = "placeholder-system-id"` back to GCS state as output.
6. `maas.lifecycle.ready` applied. `commission/ready` units polled `placeholder-system-id` forever.

**Three gates that would have caught this:**
- POST gate on `maas.lifecycle.new`: machine must exist in MaaS AND TF state must not be placeholder
- POST gate on `maas.lifecycle.commissioning`: machine must exist in MaaS after the apply
- PRE gate on `maas.lifecycle.ready`: machine must exist in MaaS AND TF must not be placeholder

### Current state of checks

| Wave | PRE | POST |
|---|---|---|
| `maas.lifecycle.new` | `maas-machines-precheck` (normalise + enroll) | **NONE** |
| `maas.lifecycle.commissioning` | `maas-lifecycle-sanity` (annihilate stuck) | **NONE** |
| `maas.lifecycle.ready` | `maas-lifecycle-sanity` | **NONE** |
| `maas.lifecycle.allocated` | `maas-lifecycle-sanity` | **NONE** |
| `maas.lifecycle.deploying` | `maas-lifecycle-sanity` | **NONE** |
| `maas.lifecycle.deployed` | `maas-lifecycle-sanity` | `maas-machines-test` (SSH port 22 only) |

Every wave after `new` has NO post-check. Nothing verifies that the apply actually produced
the expected state before handing off to the next wave.

## Table 1: All Possible Checks

| ID | Category | What | How | Notes |
|---|---|---|---|---|
| **TF-1** | TF State | `system_id` output ≠ `placeholder-system-id` | `gsutil cat gs://<bucket>/<path>/default.tfstate \| python3 -c "...check outputs.system_id..."` | Run on localhost; delegate_to from MaaS server |
| **TF-2** | TF State | `system_id` in TF state matches MaaS system_id for same hostname | Compare TF output with `maas machines read` result | Detects stale-import divergence |
| **MAAS-1** | Enrollment | Machine exists in MaaS (by hostname) | `maas <user> machines read` → check hostname present | NOT IN MAAS → always fail after new |
| **MAAS-2** | Enrollment | `power_type` in MaaS matches config | Compare MaaS `power_type` vs config value | manual≠amt → stuck commissioning |
| **MAAS-3** | Enrollment | Machine has hardware inventory (at least 1 NIC) | Check `interface_set` length > 0 | Commissioning populates this |
| **MAAS-4** | Status | `status_name` in wave-specific ALLOWED set | Check membership in per-wave allowed list | See Table 2 for per-wave lists |
| **MAAS-5** | Status | `status_name` NOT in FAILED/BROKEN set | Check NOT in `{'Failed commissioning','Failed testing','Failed deployment','Broken'}` | Always fail on error states |
| **MAAS-6** | Status | `status_name` == exact expected state | Exact equality check | Used in POST gates where state must be precise |
| **BMC-1** | Physical | AMT TCP port 16993 reachable | `nc -z -w5 <amt_addr> 16993` on MaaS server | Only for `power_type: amt` machines |
| **BMC-2** | Physical | AMT BMC power state | `amt-query.py` (wsman CIM_AssociatedPowerManagementService) | Returns `on`/`off`/`unknown` |
| **BMC-3** | Physical | Smart plug reachable | `curl http://127.0.0.1:7050/power/status?host=<plug>` | Only for `power_type: smart_plug` |
| **BMC-4** | Physical | Smart plug power state | Parse proxy JSON `{"status":"on"\|"off"}` | |
| **NET-1** | Network | SSH port 22 open on `cloud_public_ip` | `nc -z -w5 <cloud_public_ip> 22` from MaaS server | Only for deployed machines |
| **NET-2** | Network | Provisioning IP pingable from MaaS server | `ping -c1 -W3 <provisioning_ip>` | Only while machine is commissioning/deploying |

### MAAS-5 FAILED set (always fail)
`Failed commissioning`, `Failed testing`, `Failed deployment`, `Broken`

### BMC check scope
- `amt` machines: BMC-1 then BMC-2 if port open. Result = on/off/unreachable.
- `smart_plug` machines: BMC-3 then BMC-4. Result = on/off/unreachable.
- `proxmox` machines: skip BMC check (no remote BMC available).
- `manual` machines: skip BMC check.

### Annihilation (currently in maas-lifecycle-sanity, moves into PRE gate)
The existing annihilation logic (stuck Commissioning/Testing/Deploying with BMC off, OR power_type mismatch)
is PRESERVED in the PRE gate. After annihilation the gate now **FAILS** (already fixed in commit acfbd0b0).
The gate for all non-new waves absorbs and replaces `maas-lifecycle-sanity`.

## Architecture

### Approach: One shared gate playbook, thin per-wave-phase run-script wrappers

```
infra/maas-pkg/_wave_scripts/test-ansible-playbooks/maas/
  maas-lifecycle-gate/           ← shared gate playbook (new)
    playbook.yaml                ← parameterized by _MAAS_GATE_WAVE + _MAAS_GATE_MODE
    amt-query.py                 ← copied from maas-lifecycle-sanity/
    run                          ← NOT called directly; see wrappers below

  maas-lifecycle-gate-new-post/        ← test_ansible_playbook for maas.lifecycle.new
    run                                ← sets WAVE=new MODE=post; execs shared run

  maas-lifecycle-gate-commissioning-pre/   ← pre_ansible_playbook for commissioning
    run                                    ← WAVE=commissioning MODE=pre

  maas-lifecycle-gate-commissioning-post/  ← test_ansible_playbook for commissioning
    run                                    ← WAVE=commissioning MODE=post

  maas-lifecycle-gate-ready-pre/           ← pre_ansible_playbook for ready
    run

  maas-lifecycle-gate-ready-post/          ← test_ansible_playbook for ready
    run

  maas-lifecycle-gate-allocated-pre/
    run

  maas-lifecycle-gate-allocated-post/
    run

  maas-lifecycle-gate-deploying-pre/
    run

  maas-lifecycle-gate-deploying-post/
    run

  maas-lifecycle-gate-deployed-pre/        ← replaces maas-lifecycle-sanity for deployed
    run

  maas-lifecycle-gate-deployed-post/       ← replaces maas-machines-test (expanded)
    run
```

Each wrapper `run` script exports `_MAAS_GATE_WAVE` and `_MAAS_GATE_MODE`, then calls the shared
`maas-lifecycle-gate/run` which invokes `maas-lifecycle-gate/playbook.yaml`.

### Wrapper run script template
```bash
#!/bin/bash
set -euo pipefail
export _MAAS_GATE_WAVE="<wave-short-name>"   # new | commissioning | ready | allocated | deploying | deployed
export _MAAS_GATE_MODE="<mode>"              # pre | post
GATE_DIR="$(cd -- "$(dirname -- "${BASH_SOURCE[0]}")/../maas-lifecycle-gate" && pwd -P)"
exec "$GATE_DIR/run"
```

### Shared gate run script (maas-lifecycle-gate/run)
Same boilerplate as existing run scripts (source set_env.sh, _activate_ansible_locally,
ansible-galaxy, ansible-playbook with inventory).

## Files to Create / Modify

### `infra/maas-pkg/_wave_scripts/test-ansible-playbooks/maas/maas-lifecycle-gate/playbook.yaml` — create

The main gate playbook. Structure:

**Play 1 (localhost)** — load config, build machine list, load TF state facts

Tasks:
1. `Load config_base role`
2. `Load framework.yaml` (GCS bucket)
3. `capture-config-fact.yaml` (MaaS admin user)
4. `Set gate parameters` — set_fact from `lookup('env', '_MAAS_GATE_WAVE')` and `lookup('env', '_MAAS_GATE_MODE')`
5. `Build machine list` — same Jinja2 as existing (maas.lifecycle.new wave, skip inheritance)
6. `Check TF state for each machine (TF-1, TF-2)` — gsutil cat + python3 parse; runs on localhost

For the TF state check (Play 1, delegated localhost task):
```python
# Read gs://<bucket>/<machine_path>/default.tfstate
# Parse outputs.system_id.value
# Check != "placeholder-system-id"
# Store in dict: {hostname: {tf_system_id: "...", tf_placeholder: true/false}}
```

**Play 2 (maas_region)** — read MaaS state, run BMC checks, evaluate gates

Tasks:
1. `Login to MaaS CLI`
2. `Fail if login failed`
3. `Read all machines from MaaS` — `maas <user> machines read`
4. `Build _maas_state map` — {hostname: {system_id, status_name, power_type, interface_set_count}}
5. `=== Current MaaS States ===` — debug table showing all machines
6. `Warn about Broken state machines`
7. `[PRE only] Restore AMT standby via smart plug before BMC check` (from maas-lifecycle-sanity)
8. `[PRE only] Query BMC power state for AMT machines`
9. `[PRE only] Record AMT power states`
10. `[PRE only] Merge AMT states into _bmc_state`
11. `[PRE only] Query power state for smart-plug machines`
12. `[PRE only] Merge smart-plug states into _bmc_state`
13. `[PRE only] Determine which machines to annihilate` (power_type mismatch OR stuck transitional with BMC off)
14. `[PRE only] === Annihilation list ===`
15. `[PRE only] All machines OK — no annihilation needed`
16. `[PRE only] Prompt operator before annihilating` (confirm mode)
17. `[PRE only] Delete annihilated machines from MaaS`
18. `[PRE only] Wipe GCS Terraform state for annihilated machines`
19. `[PRE only] Report annihilation complete`
20. `[PRE only] FAIL after annihilation` ← the fix from commit acfbd0b0
21. `Evaluate gate checks` — Jinja2 block that applies the check table for (_wave, _mode)
22. `=== Gate Results ===` — debug: pass/fail per check per machine
23. `Fail if any check failed`

**Gate evaluation Jinja2 logic (task 21):**

```jinja2
{%- set failures = [] -%}
{%- for m in _machines -%}
  {%- set ms = _maas_state.get(m.hostname, {}) -%}
  {%- set tf = hostvars['localhost']['_tf_state'].get(m.hostname, {}) -%}

  {# TF-1: placeholder check #}
  {%- if _gate_checks.tf1 and tf.get('tf_placeholder', true) -%}
    {%- set _ = failures.append(m.hostname ~ ': TF-1 FAIL — system_id is placeholder (machine not enrolled or state wiped)') -%}
  {%- endif -%}

  {# TF-2: TF matches MaaS #}
  {%- if _gate_checks.tf2 and ms -%}
    {%- if tf.get('tf_system_id','') != ms.get('system_id','') -%}
      {%- set _ = failures.append(m.hostname ~ ': TF-2 FAIL — TF system_id=' ~ tf.get('tf_system_id','?') ~ ' MaaS system_id=' ~ ms.get('system_id','?')) -%}
    {%- endif -%}
  {%- endif -%}

  {# MAAS-1: enrolled #}
  {%- if _gate_checks.maas1 and not ms -%}
    {%- set _ = failures.append(m.hostname ~ ': MAAS-1 FAIL — not in MaaS') -%}
  {%- endif -%}

  {# MAAS-2: power_type #}
  {%- if _gate_checks.maas2 and ms -%}
    {%- if ms.get('power_type','') != m.power_type -%}
      {%- set _ = failures.append(m.hostname ~ ': MAAS-2 FAIL — power_type MaaS=' ~ ms.get('power_type','?') ~ ' config=' ~ m.power_type) -%}
    {%- endif -%}
  {%- endif -%}

  {# MAAS-3: hardware inventory #}
  {%- if _gate_checks.maas3 and ms -%}
    {%- if ms.get('interface_count', 0) == 0 -%}
      {%- set _ = failures.append(m.hostname ~ ': MAAS-3 FAIL — no NICs in MaaS hardware inventory (commissioning never ran?)') -%}
    {%- endif -%}
  {%- endif -%}

  {# MAAS-4: allowed states #}
  {%- if _gate_checks.maas4_allowed and ms -%}
    {%- if ms.get('status_name','') not in _gate_checks.maas4_allowed -%}
      {%- set _ = failures.append(m.hostname ~ ': MAAS-4 FAIL — status=' ~ ms.get('status_name','?') ~ ' not in allowed=' ~ (_gate_checks.maas4_allowed | join(','))) -%}
    {%- endif -%}
  {%- endif -%}

  {# MAAS-5: not failed/broken #}
  {%- set failed_states = ['Failed commissioning','Failed testing','Failed deployment','Broken'] -%}
  {%- if _gate_checks.maas5 and ms -%}
    {%- if ms.get('status_name','') in failed_states -%}
      {%- set _ = failures.append(m.hostname ~ ': MAAS-5 FAIL — status=' ~ ms.get('status_name','?') ~ ' (error state)') -%}
    {%- endif -%}
  {%- endif -%}

  {# MAAS-6: exact state #}
  {%- if _gate_checks.maas6_exact and ms -%}
    {%- if ms.get('status_name','') != _gate_checks.maas6_exact -%}
      {%- set _ = failures.append(m.hostname ~ ': MAAS-6 FAIL — expected exactly ' ~ _gate_checks.maas6_exact ~ ' got ' ~ ms.get('status_name','?')) -%}
    {%- endif -%}
  {%- endif -%}

  {# BMC-on: machine should be powered on #}
  {%- if _gate_checks.bmc_on and m.power_type in ['amt','smart_plug'] -%}
    {%- set bmc = _bmc_state.get(m.hostname, {}) -%}
    {%- if bmc.get('power_state','unknown') in ['off','unknown'] -%}
      {%- set _ = failures.append(m.hostname ~ ': BMC-on FAIL — power_state=' ~ bmc.get('power_state','unknown') ~ ' (should be on)') -%}
    {%- endif -%}
  {%- endif -%}

  {# NET-1: SSH port 22 #}  [checked in separate task — see below]

{%- endfor -%}
{{ failures }}
```

### TF state reader (Play 1, localhost)

Add a task to Play 1 that reads GCS Terraform state for each machine and extracts
`system_id` from the outputs:

```yaml
- name: Read TF state from GCS for each machine (TF-1, TF-2)
  ansible.builtin.shell: |
    python3 << 'PYEOF'
    import subprocess, json, sys

    bucket = "{{ _tf_bucket }}"
    results = {}

    machines = {{ _all_machines | to_json }}
    for m in machines:
        path = m['machine_path'] + '/default.tfstate'
        url = f"gs://{bucket}/{path}"
        r = subprocess.run(['gsutil', '-q', 'cat', url], capture_output=True, text=True)
        if r.returncode != 0:
            results[m['hostname']] = {'tf_system_id': '', 'tf_placeholder': True, 'tf_missing': True}
            continue
        try:
            state = json.loads(r.stdout)
            sid = state.get('outputs', {}).get('system_id', {}).get('value', '')
            results[m['hostname']] = {
                'tf_system_id': sid,
                'tf_placeholder': sid == 'placeholder-system-id' or sid == '',
                'tf_missing': False
            }
        except Exception as e:
            results[m['hostname']] = {'tf_system_id': '', 'tf_placeholder': True, 'tf_missing': True}

    print(json.dumps(results))
    PYEOF
  register: _tf_state_raw
  changed_when: false

- name: Set TF state fact
  ansible.builtin.set_fact:
    _tf_state: "{{ _tf_state_raw.stdout | from_json }}"
```
