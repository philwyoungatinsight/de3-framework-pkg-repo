# Plan: Remote Package System

## Objective

Enable packages to come from external git repos. A user can register an external repo as a
"package repository", browse its public packages, and import individual packages into this
project.

**Responsibility split:**
- **GUI** — reads and writes the two config YAML files (`framework_package_repositories.yaml`,
  `framework_packages.yaml`) directly. Users can add/edit/remove repos and package entries,
  modify any package parameter (`public`, `version`, `source`, `repo`, `import_path`), etc.
  After each config write the GUI calls `pkg-mgr sync` to apply the changes.
- **`pkg-mgr`** — owns all filesystem operations: cloning repos into `_ext_packages/`,
  creating/removing `infra/<pkg>` symlinks, running collision checks. It is the single source
  of truth for "what the filesystem should look like given the current config".

Symlinks are committed to git; `_ext_packages/` (the cloned repos) is gitignored.

---

## Context

### What already exists

- **`config/framework_packages.yaml`** — lists all packages with `name`, `public`, and stub
  `source`/`version` fields. The `source:` field already hints at the intended design.
- **`_ext_packages/<repo-name>/`** at repo root — existing clone destination. Git-ignored.
  Populated when "+ Repo" is used in the GUI.
- **`_scan_packages()`** (homelab_gui.py:3029) — scans `infra/<pkg>/` (built-in) AND
  `_ext_packages/<repo>/infra/<pkg>/` (external clones).
- **`clone_pkg_repo()` / `remove_pkg_repo()`** (7700–7762) — async handlers that clone/remove
  whole repos; state stored in `state/current.yaml`.
- **`packages_view()` / `_add_pkg_repo_dialog()`** (12146–12342) — current "+ Repo" UI.

### Resolved design decisions

1. **Symlinks committed to git** — `infra/<pkg-name>` symlinks pointing into `_ext_packages/`
   are tracked in git. After a fresh clone, symlinks are dangling until `pkg-mgr sync` is run
   to re-clone the repos.
2. **Unit file collision** — a collision is when importing a package would result in two
   directories under `infra/` that both contain a `terragrunt.hcl` file at the same relative
   path. Check by walking the candidate package's `_stack/` tree for `terragrunt.hcl` files
   and comparing paths against all `terragrunt.hcl` files already present under `infra/`.
3. **`pkg-mgr` script owns all filesystem operations** — cloning, symlinking, unlinking,
   collision detection. The GUI never does git operations or creates symlinks directly.
4. **GUI is the config editor** — directly reads and writes `framework_packages.yaml` and
   `framework_package_repositories.yaml`. After any config change, calls `pkg-mgr sync` to
   reconcile the filesystem with the current config state.

### Critical gap closed by this plan

`root.hcl` only resolves units under `infra/<pkg>/`. Imported packages get a symlink at
`infra/<pkg-name>` → `../_ext_packages/<repo-slug>/infra/<pkg-name>/`, making them
transparent to root.hcl.

### `config/framework_packages.yaml` — **modify**

Add two new optional fields to the schema. Two distinct import cases are now documented:

**Case A — package lives inside a registered parent repo (no `source:`):**
```yaml
  - name: some-pkg
    public: false
    repo: community-infra-pkgs   # NEW: parent repo slug (matches framework_package_repositories entry)
    import_path: some-pkg        # NEW: subdir inside parent repo's infra/ (defaults to name)
```

**Case B — package has its own source git URL (either defined locally or inherited from
the parent repo's `framework_packages.yaml`):**
```yaml
  - name: external-foo-pkg
    public: false
    source: https://github.com/foo/bar.git   # existing field — the package's own git repo
    version: '>=1.0.0'                        # existing field
    repo: bar                                 # NEW: slug derived from source URL (last path component, no .git)
    import_path: external-foo-pkg             # NEW: subdir inside bar/infra/ (defaults to name)
```

When `source:` is set, `repo:` is always derived from it — it is the slug for the clone
directory at `_ext_packages/<slug>/`. The parent repo (if any) is not stored in the entry;
`repo:` always points to where the package actually lives.

**Slug derivation rule:** strip `.git`, take the last `/`-separated component.
Examples: `https://github.com/foo/bar.git` → `bar`, `git@github.com:org/baz` → `baz`.
If `_ext_packages/<slug>/` already exists and is a different URL, abort with an error.

No other existing entries are changed.

### `scripts/human-only-scripts/pkg-mgr/run` — **create**

Central script that owns all package lifecycle operations. The GUI invokes this after
updating the YAML config files.

**Commands:**

```
pkg-mgr sync                    # apply current framework_packages.yaml state:
                                #   for each entry with repo: set —
                                #     if source: present → clone from source URL if missing
                                #     else → parent repo must already be cloned
                                #   create missing infra/<pkg> symlinks,
                                #   remove symlinks for packages no longer in config
pkg-mgr add-repo <url> [<name>] # register a package repository:
                                #   clone it, read its framework_packages.yaml,
                                #   add entry to framework_package_repositories.yaml
pkg-mgr remove-repo <name>      # deregister a repo:
                                #   remove all imported packages from that repo,
                                #   delete _ext_packages/<name>/, remove repo entry
pkg-mgr import <repo> <pkg>     # import one package from a registered repo:
                                #   read _ext_packages/<repo>/config/framework_packages.yaml
                                #   find <pkg> entry; if it has source: → derive slug from URL,
                                #     clone source URL to _ext_packages/<slug>/ if missing,
                                #     set repo: <slug> and source: <url> in local entry
                                #   else → set repo: <repo> (parent), no source: in local entry
                                #   run collision check (fail on unit-collision)
                                #   create infra/<pkg> symlink
                                #   add entry to framework_packages.yaml
pkg-mgr remove <pkg>            # remove one imported package:
                                #   delete infra/<pkg> symlink,
                                #   remove entry from framework_packages.yaml
                                #   (does NOT delete the clone — other pkgs may share it)
pkg-mgr list-remote <repo>      # list public packages in a registered repo; for each show:
                                #   name, description, already_imported flag,
                                #   and source: URL if the package has its own repo
pkg-mgr check <repo> <pkg>      # run collision check and print results; exit 0=clean, 1=collision
                                #   resolves source: URL the same way import does
pkg-mgr status                  # print table: all imported packages, their repo/source, symlink state
```

**Script structure:**

```bash
#!/usr/bin/env bash
set -euo pipefail
source "$(git rev-parse --show-toplevel)/set_env.sh"

GIT_ROOT="$(git rev-parse --show-toplevel)"
EXT_PACKAGES_DIR="$GIT_ROOT/_ext_packages"
INFRA_DIR="$GIT_ROOT/infra"
PKG_REPOS_CFG="$GIT_ROOT/config/framework_package_repositories.yaml"
FRAMEWORK_PKGS_CFG="$GIT_ROOT/config/framework_packages.yaml"

CMD="${1:-}"
shift || true

case "$CMD" in
  sync)        _cmd_sync "$@" ;;
  add-repo)    _cmd_add_repo "$@" ;;
  remove-repo) _cmd_remove_repo "$@" ;;
  import)      _cmd_import "$@" ;;
  remove)      _cmd_remove "$@" ;;
  list-remote) _cmd_list_remote "$@" ;;
  check)       _cmd_check "$@" ;;
  status)      _cmd_status "$@" ;;
  *)           echo "Unknown command: $CMD"; exit 1 ;;
esac
```

**Collision check logic (`_cmd_check` / shared helper `_check_collisions`):**

Both `check` and `import` call `_resolve_pkg_repo` first to handle the `source:` indirection,
then run two independent checks:

```bash
_resolve_pkg_repo() {
  # Given a parent repo slug and package name, return the actual clone slug and import path.
  # If the package entry in the parent repo has source:, derive slug from that URL and
  # ensure it is cloned. Outputs: "<actual_slug> <import_path>"
  local parent_repo="$1" pkg_name="$2"
  local parent_cfg="$EXT_PACKAGES_DIR/$parent_repo/config/framework_packages.yaml"

  local source_url import_path
  source_url=$(python3 -c "
import yaml, sys
pkgs = yaml.safe_load(open('$parent_cfg'))['framework_packages']
entry = next((p for p in pkgs if p['name'] == '$pkg_name'), {})
print(entry.get('source', ''))
")
  import_path=$(python3 -c "
import yaml, sys
pkgs = yaml.safe_load(open('$parent_cfg'))['framework_packages']
entry = next((p for p in pkgs if p['name'] == '$pkg_name'), {})
print(entry.get('import_path', '$pkg_name'))
")

  if [[ -n "$source_url" ]]; then
    local slug
    slug=$(_url_to_slug "$source_url")
    _ensure_cloned "$slug" "$source_url"
    echo "$slug $import_path"
  else
    echo "$parent_repo $import_path"
  fi
}
```

Two independent checks — both run, all findings returned:

**Check 1 — Unit file collision:**
Two packages would place a `terragrunt.hcl` at the same path under `infra/`.

```bash
_check_unit_collisions() {
  local repo="$1" pkg_import_path="$2"
  local candidate_dir="$EXT_PACKAGES_DIR/$repo/infra/$pkg_import_path"

  # Walk candidate _stack/ for all terragrunt.hcl files
  while IFS= read -r candidate_tg; do
    rel="${candidate_tg#$candidate_dir/}"
    candidate_rel="$pkg_import_path/$rel"
    # Check same relative path among all existing infra/ units
    while IFS= read -r existing_tg; do
      existing_rel="${existing_tg#$INFRA_DIR/}"
      if [[ "$existing_rel" == "$candidate_rel" ]]; then
        echo "unit-collision: '$candidate_rel' already exists as '$existing_rel'"
      fi
    done < <(find "$INFRA_DIR" -name "terragrunt.hcl" -not -path "*/.terragrunt-cache/*" \
                                                        -not -path "*/_ext_packages/*")
  done < <(find "$candidate_dir/_stack" -name "terragrunt.hcl" 2>/dev/null)
}
```

**Check 2 — config_params collision:**
The remote package's `_config/<pkg>.yaml` and any existing package's `_config/<pkg>.yaml`
both define the same `config_params` key path but with different values. This causes a
silent overwrite on deep-merge.

Implemented as a Python helper called inline from bash (YAML parsing in bash is not viable):

```bash
_check_config_collisions() {
  local repo="$1" pkg_import_path="$2"
  local candidate_cfg="$EXT_PACKAGES_DIR/$repo/infra/$pkg_import_path/_config/$pkg_import_path.yaml"
  [[ -f "$candidate_cfg" ]] || return 0

  python3 - "$candidate_cfg" "$INFRA_DIR" <<'PYEOF'
import sys, yaml, pathlib, itertools

def flatten(d, prefix=""):
    """Yield (dot-path, value) for all leaf nodes in a nested dict."""
    if isinstance(d, dict):
        for k, v in d.items():
            yield from flatten(v, f"{prefix}.{k}" if prefix else str(k))
    else:
        yield prefix, d

candidate_cfg = pathlib.Path(sys.argv[1])
infra_dir     = pathlib.Path(sys.argv[2])

candidate_data = yaml.safe_load(candidate_cfg.read_text()) or {}
candidate_flat = dict(flatten(candidate_data))

for existing_cfg in infra_dir.rglob("_config/*.yaml"):
    if ".terragrunt-cache" in str(existing_cfg):
        continue
    if existing_cfg == candidate_cfg:
        continue
    existing_data = yaml.safe_load(existing_cfg.read_text()) or {}
    existing_flat = dict(flatten(existing_data))
    for path, cval in candidate_flat.items():
        if path in existing_flat and existing_flat[path] != cval:
            print(f"config-collision: '{path}' = {cval!r} in candidate, "
                  f"{existing_flat[path]!r} in {existing_cfg.relative_to(infra_dir)}")
PYEOF
}
```

**Combined `_check_collisions`:**

```bash
_check_collisions() {
  local repo="$1" pkg_import_path="$2"
  _check_unit_collisions   "$repo" "$pkg_import_path"
  _check_config_collisions "$repo" "$pkg_import_path"
}
```

`pkg-mgr check` calls `_check_collisions`, prints all lines, exits 0 if no output else 1.
Lines prefixed `unit-collision:` are hard errors; `config-collision:` are warnings (shown
in the GUI but do not block import — the user must acknowledge them explicitly).

**Symlink creation (`_create_symlink`):**

```bash
_create_symlink() {
  local pkg_name="$1" repo="$2" import_path="$3"
  local link="$INFRA_DIR/$pkg_name"
  local target="../_ext_packages/$repo/infra/$import_path"

  if [[ -d "$link" && ! -L "$link" ]]; then
    echo "ERROR: $link is a real directory (built-in package). Cannot import as '$pkg_name'."
    exit 1
  fi
  ln -sfn "$target" "$link"
  echo "Symlink: infra/$pkg_name -> $target"
}
```

**Slug derivation helper (`_url_to_slug`):**

```bash
_url_to_slug() {
  # Strip trailing /, remove .git suffix, take last path component
  local url="${1%/}"
  url="${url%.git}"
  echo "${url##*/}"
}
```

Used by both `_cmd_import` and `_cmd_sync` when a package entry has `source:` set.

**`_ensure_cloned(slug, url)`** — shared helper used by `import` and `sync`:

```bash
_ensure_cloned() {
  local slug="$1" url="$2"
  local dest="$EXT_PACKAGES_DIR/$slug"
  if [[ -d "$dest" ]]; then
    # Verify it was cloned from the same URL (check remote.origin.url)
    local existing_url
    existing_url=$(git -C "$dest" remote get-url origin 2>/dev/null || true)
    if [[ -n "$existing_url" && "$existing_url" != "$url" ]]; then
      echo "ERROR: _ext_packages/$slug/ exists but points to '$existing_url', not '$url'"
      exit 1
    fi
    return 0  # already present
  fi
  echo "Cloning $url → _ext_packages/$slug/ ..."
  git clone --depth=1 "$url" "$dest"
}
```

**YAML reads/writes:** Use `python3 -c` inline or a small helper Python one-liner rather than
trying to parse YAML in bash. Pattern:

```bash
_yaml_get() { python3 -c "import yaml,sys; d=yaml.safe_load(sys.stdin); print(d$1 or '')" < "$2"; }
_yaml_set_list() { ... }  # write via Python yaml.dump to tmp file then mv
```

No shell redirect (`>`) on YAML files — always write to a `.tmp` and rename atomically.
