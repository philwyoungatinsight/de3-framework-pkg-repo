# Plan: Watchdog Report File

## Objective
Update the `/watchdog` skill so that every watchdog run (both the initial invocation and
each periodic cron-triggered agent) writes a structured YAML report to
`$_DYNAMIC_DIR/watchdog-report/watchdog_report.yaml`, archiving any previous report
alongside it. If the report concludes that user input is needed, the running watchdog
session asks the user directly.

## Context

### Existing skill: `.claude/commands/watchdog.md`
Three steps: (1) check for existing cron job, (2) create if absent, (3) run one immediate
check via `bash .../check` and show last 3 log lines.

The cron prompt (inside `CronCreate`) currently says:
> "Run the build watchdog check immediately … If build=STOPPED and a build was supposed
> to be running, … report. If build=RUNNING, report … Do NOT just report and go silent
> — if something is wrong, fix it."

There is no report file today — state is only appended to `~/.build-watchdog.log`.

### Dynamic dir
`$_DYNAMIC_DIR` = `/home/pyoung/git/pwy-home-lab/config/tmp/dynamic`  
Report path: `$_DYNAMIC_DIR/watchdog-report/watchdog_report.yaml`  
Archive dir: `$_DYNAMIC_DIR/watchdog-report/archived/`

### Timestamp format (ai-log convention)
`$(date +%Y%m%d%H%M%S)` — e.g. `20260416123045`

### Report YAML structure
```yaml
timestamp: "2026-04-16T12:30:45"
build_status: "RUNNING"          # RUNNING | STOPPED | UNKNOWN
assessment: "..."                # one-line summary
description: |
  Multi-line prose: what wave is active, last error seen, MaaS machine states, etc.
user_input_needed: false
user_input_message: ""           # empty when user_input_needed: false
```

`user_input_needed: true` situations:
- Build is STOPPED but it was expected to be running (no clean completion line in log)
- A wave has been failing for >2 consecutive checks (same ERROR line repeated)
- MaaS is unreachable for >2 checks in a row

### `.claude/commands/watchdog.md` — modify

Add a new **Step 4 — Write watchdog report** after the existing Step 3.

The step instructs the agent to:
1. Determine `DYNAMIC_DIR` — hardcode the known path
   `GIT_ROOT/config/tmp/dynamic` (same as `$_DYNAMIC_DIR`, but resolved so the
   agent doesn't need to source set_env.sh):
   ```
   REPORT_DIR=/home/pyoung/git/pwy-home-lab/config/tmp/dynamic/watchdog-report
   REPORT_FILE=$REPORT_DIR/watchdog_report.yaml
   ARCHIVE_DIR=$REPORT_DIR/archived
   ```
2. If `watchdog_report.yaml` already exists, move it to
   `$ARCHIVE_DIR/$(date +%Y%m%d%H%M%S)_watchdog_report.yaml` before writing the new one.
3. Read `~/.build-watchdog.log` (last 10 lines) and the most recent wave log tail to
   synthesise `assessment` and `description`.
4. Write the YAML report using the Write tool (not bash redirect — Write tool is atomic).
5. Evaluate `user_input_needed`: set `true` if:
   - Build is STOPPED but last log line does not contain `"build complete"` or `"clean complete"`
   - Same ERROR string appears in 3 consecutive log lines (stuck wave)
   - MaaS shows `maas-unreachable` in the last log line
6. If `user_input_needed: true`: output the `user_input_message` to the user as a direct
   question and wait for their response before ending the session.

Also update the **cron prompt** (the multi-line string passed to `CronCreate`) to append
the same "Write watchdog report" instructions at the end, so every periodic cron-triggered
agent also produces a report.
