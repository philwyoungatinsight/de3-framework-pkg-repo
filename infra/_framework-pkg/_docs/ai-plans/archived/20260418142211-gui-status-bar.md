# Plan: Persistent Status Bar in GUI

## Objective

Add a "Show status bar" checkbox to the Appearance → Layout section. When enabled, a
fixed-height strip is permanently reserved at the bottom of the app. The existing GCS
sync / inventory messages write into that strip instead of floating over content. While
an activity is in progress the strip is highlighted; when idle it is empty but still
occupies space, so nothing (e.g. the bottom line of the terminal) is ever obscured.

When the status bar is disabled, the app falls back to the existing `startup_status_banner()`
floating-overlay behaviour.

## Context

Key findings:

- **`startup_status_banner()`** (line ~16099) renders `position="fixed", bottom="0"` — it
  overlaps content rather than reserving space. It is driven entirely by `app_status_message`.
- **`index()`** (line ~16586): root `rx.box` contains `backend_error_banner()`,
  `startup_status_banner()`, `navbar()`, then the main-area `rx.cond`. The root box has
  no explicit flex layout; heights inside are hardcoded (`100vh`, `calc(100vh - 36px)`).
- **Layout section** (lines 14315–14361) already has "Floating panels mode" + "Drag Width"
  slider. A new checkbox fits directly below "Floating panels mode".
- **Pattern for toggles**: `toggle_X` accepts a bool (from `rx.checkbox`),
  `flip_X` takes no arg (for row-click). Both set the same state var.
- **`_appearance_menu_item()`** (line 13819) takes `(label, checked_var, on_change, on_row_click)`.
- **`gcs_unit_sync_running` / `gcs_wave_sync_running`** are consumed only by
  `app_status_message`. `app_status_message` returns `""` when idle.

#### 4. Add `static_status_bar()` component function

Add just before `startup_status_banner()` (around line 16099):

```python
_STATUS_BAR_HEIGHT = "26px"


def static_status_bar() -> rx.Component:
    """Permanent bottom strip — always occupies space; highlighted when active.

    Used when AppState.show_status_bar is True. Content mirrors app_status_message.
    Caller is responsible for gating on show_status_bar via rx.cond.
    """
    return rx.box(
        rx.cond(
            AppState.app_activity_active,
            # Active: spinner + message, accent background
            rx.hstack(
                rx.spinner(size="1", color="white"),
                rx.text(
                    AppState.app_status_message,
                    font_size="12px",
                    font_weight="500",
                    color="white",
                ),
                align="center",
                spacing="2",
                height="100%",
            ),
            # Idle: empty, no content
            rx.box(),
        ),
        width="100%",
        height=_STATUS_BAR_HEIGHT,
        flex_shrink="0",
        border_top="1px solid var(--gui-border)",
        background=rx.cond(
            AppState.app_activity_active,
            "var(--accent-9)",
            "var(--gui-surface)",
        ),
        padding_x="16px",
        display="flex",
        align_items="center",
        justify_content="center",
        transition="background 0.2s ease",
    )
```

#### 5. Convert `index()` root box to flex column; thread status bar into layout

**Problem**: `startup_status_banner()` uses `position="fixed"` so it overlaps content.
To give the status bar *real* reserved space we need the root `rx.box` to be a flex
column so `flex="1"` / `flex_shrink="0"` can govern heights.

**Change 5a — root box**: Add flex props.

```python
# Before (line ~16587):
def index() -> rx.Component:
    return rx.box(
        rx.el.style(_GUI_THEME_CSS),
        backend_error_banner(),
        startup_status_banner(),
        navbar(),
        rx.cond(   # main area
            ...
        ),
        ...
    )

# After:
def index() -> rx.Component:
    return rx.box(
        rx.el.style(_GUI_THEME_CSS),
        backend_error_banner(),                           # fixed, no flow impact
        # Floating overlay banner — only when status bar is off
        rx.cond(
            ~AppState.show_status_bar,
            startup_status_banner(),
            rx.fragment(),
        ),
        navbar(),                                         # flex child, natural height
        # Main content — fills remaining space
        rx.box(
            rx.cond(                                      # floating / normal / maximized
                AppState.floating_panels_mode,
                ...floating mode box (height="100%")...,
                rx.cond(
                    AppState.maximized_panel == "",
                    ...normal mode box (height="100%")...,
                    ...maximized mode box (height="100%")...,
                ),
            ),
            flex="1",
            min_height="0",
            overflow="hidden",
            width="100%",
        ),
        # Permanent status bar — only when enabled
        rx.cond(
            AppState.show_status_bar,
            static_status_bar(),
            rx.fragment(),
        ),
        display="flex",
        flex_direction="column",
        height="100vh",
        overflow="hidden",
    )
```

**Change 5b — internal height values**: With the parent flex managing the height, all
three inner layout boxes must change from `height="100vh"` / `height="calc(100vh - 36px)"`
to `height="100%"`.

Exact replacements:

| Location | Old value | New value |
|---|---|---|
| Floating mode `#left-column` box style (line ~16606) | `"height": "calc(100vh - 36px)"` | `"height": "100%"` |
| Normal mode `#main-panels` box (line ~16679) | `height="100vh"` | `height="100%"` |
| Maximized mode box (line ~16695) | `height="100vh"` | `height="100%"` |

> **Why `100%` works now**: the wrapping `rx.box` with `flex="1"` / `min_height="0"`
> expands to fill exactly the space between navbar and status bar. Child boxes using
> `height="100%"` fill that container. Previously they used `100vh` and relied on
> overflow clipping — that approach breaks with a fixed-height flex sibling.

2. **Status bar on (default)**: a thin strip appears at the bottom of the app at all

   popup banner reappears during GCS syncs.
4. **Toggle back on**: strip returns; no layout jump or overflow.
5. **Maximized panel**: strip still reserves space at bottom; panel content not cut off.
6. **Floating panels mode + status bar on**: check that the left sidebar height is correct
   (fills to the strip, not to the bottom of the viewport).
