# fw-repo-labels — Add Label System to fw-repos-visualizer

**Date**: 2026-04-25
**Plan**: `infra/_framework-pkg/_docs/ai-plans/archived/20260425152008-fw-repo-labels.md`

## What was done

Added a label system for framework repos in `framework_repo_manager.yaml`. Labels are a list
of YAML objects with a required `name` field and optional qualifiers. Reserved labels use
underscore prefix (`_purpose`, `_owner`, `_docs`). Initialized `_purpose` and `_docs` for
all 12 repos in `pwy-home-lab-pkg`'s `framework_repo_manager.yaml`.

**Changes in de3-runner (`a8fb7cd` + `b75b717` + `406dfc9`):**
- `scanner.py`: read `labels` from `framework_repo_manager.yaml` entries; back-fill into
  scan result entries; merged merge semantics (non-empty wins) so template repos can't wipe
  labels set by the deployment repo; moved back-fill before the `path.exists()` early return
  so inaccessible repos still receive their declared labels
- `renderer.py`: text renderer shows value-carrying labels under the repo header line;
  DOT renderer adds `_purpose` to the cluster subtitle
- `framework_repo_manager.yaml` (de3-runner template): commented out `proxmox-pkg-repo`
  example entry that was live despite a comment saying it was commented out — was causing
  the phantom repo to appear in the diagram and wiping labels via last-write-wins
- `_framework-pkg.yaml`: bumped `_provides_capability` to `1.12.0`
- `version_history.md`: added `1.12.0` entry

**Changes in pwy-home-lab-pkg (`framework_repo_manager.yaml`):**
- Added `labels: [_purpose, _docs]` to all 12 framework_repos entries with descriptions
  derived from package READMEs

## Bugs fixed along the way

1. **Template overwrite bug**: `_load_repo_manager` used last-write-wins semantics, so when
   de3-runner's template (scanned via BFS) re-declared `proxmox-pkg-repo` with no labels,
   it wiped the labels pwy-home-lab-pkg had set. Fixed with non-empty-wins merge.

2. **Early-return bug**: The notes/labels back-fill in `_scan_dir` was positioned after the
   `if not path.exists(): return` guard. Repos whose upstream clone fails (e.g. inaccessible
   GitHub repos) returned early and never got their declared metadata. Moved back-fill before
   the guard.

3. **Live example entry**: de3-runner's template `framework_repo_manager.yaml` had
   `proxmox-pkg-repo` listed as an "Example A" entry with a comment saying it was "commented
   out", but it was actually live. This violated CLAUDE.md's "no placeholder examples" rule
   and caused the repo to appear as a phantom with `created_by: de3-runner` attribution.
   Commented it out properly.

## Verification

```
fw-repos-visualizer --refresh && fw-repos-visualizer --list --format text
```

All 12 repos show `[_purpose]` and `[_docs]` labels in the text output. Labels are present in
`known-fw-repos.yaml` for all repos including inaccessible ones (proxmox-pkg-repo).
