### Phase 2 — 6.49.19 → 7.22.1 (this session)
**Blockers hit and fixed:**

1. **STABLE URL 404**: `https://download.mikrotik.com/routeros/STABLE/routeros-STABLE-arm.npk` does not exist.
   - Fix: use versioned URL `https://download.mikrotik.com/routeros/7.22.1/routeros-7.22.1-arm.npk`
   - Latest stable as of 2026-04-14 is 7.22.1

2. **`/system reboot` CLI confirmation prompt**: `community.routeros.command` (network_cli mode) times out
   after 300s waiting for RouterOS `>` prompt, but RouterOS shows `[y/N]:` confirmation that the
   module never answers.
   - Fix: replaced CLI reboot with Python paramiko `exec_command('/system reboot')` (no PTY = no
     confirmation prompt). Set as delegate_to localhost in the playbook.

3. **`api-ssl` TLS handshake failure**: RouterOS 7 ships with `api-ssl` service having `certificate: none`.
   Even with `_provider_routeros_insecure: true`, RouterOS needs a cert to initiate TLS.
   - Fix: create self-signed cert via `/certificate add/sign` + `/ip service set api-ssl certificate=...`
   - Added this as Phase D cleanup step in `upgrade-to-7.yaml` (idempotent)
