# Rocky Linux 10 MaaS import pipeline + ms01-02 Rocky+OVS config

## Summary

Automated the Rocky Linux 10 GenericCloud → MaaS custom boot resource import
(same pattern as Debian Trixie). Configured ms01-02 to deploy Rocky 10 via MaaS
with an OVS bridge declared for its 10G NIC. Updated `rocky_images` config to use
the structured format expected by the import pipeline.

## Changes

### New: `import-rocky-image.yaml` (both configure-region and configure-server)

- `infra/maas-pkg/_tg_scripts/maas/configure-region/tasks/import-rocky-image.yaml`
- `infra/maas-pkg/_tg_scripts/maas/configure-server/tasks/import-rocky-image.yaml`

Mirrors the existing `import-debian-image.yaml` pipeline but for Rocky Linux:
1. Downloads Rocky GenericCloud qcow2 to /tmp via wget
2. Connects via qemu-nbd; dynamically detects root (xfs/ext4) and EFI (vfat)
   partitions using `blkid` — no hardcoded partition numbers
3. Chroot customization:
   - Fixes resolv.conf (removes systemd-resolved symlink)
   - `dnf install grub2-efi-x64 grub2-efi-x64-modules grub2-tools efibootmgr
     openssh-server cloud-init`
   - Tries EPEL install → `dnf install netplan` (MaaS post-deploy runs `netplan info`)
   - Falls back to a stub `/usr/local/bin/netplan` that returns `features: []`
     if EPEL/netplan unavailable (EPEL 10 is still in beta for Rocky 10)
   - Runs `grub2-install --target=x86_64-efi --efi-directory=/boot/efi
     --bootloader-id=rocky --removable --no-nvram`
   - Creates `/curtin` marker directory (required by curtin write_image_to_disk)
4. Disconnects nbd, converts qcow2 → raw → gzip (ddgz format, same as Debian)
5. Copies to /var/snap/maas/common/rocky-import/ and imports via
   `maas maas-admin boot-resources create ... filetype=ddgz`
6. Includes rescue block that tears down nbd mounts on failure

### Updated: `import-maas-images.yaml` (both paths)

Replaced the Rocky placeholder `debug: msg` block with a real import block
(mirrors the Debian import block):
- Calls `import-rocky-image.yaml` in a loop over `_maas_config.rocky_images`
- Waits for boot resources to reach `Uploaded` state
- Reports final status
- Updated file header comment to document the new `rocky_images` dict format

**`rocky_images` list** — restructured from the informational `[name, url]` format
to the import-pipeline format matching `debian_images`:
