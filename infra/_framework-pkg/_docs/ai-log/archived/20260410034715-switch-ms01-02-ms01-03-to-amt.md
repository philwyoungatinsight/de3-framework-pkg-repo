# Switch ms01-02 and ms01-03 to AMT Power Type

**Date:** 2026-04-10  
**Commit:** dcaf0f1

## Summary

Switched ms01-02 and ms01-03 from `smart_plug` (Tapo) to `amt` (Intel Active
Management Technology) power control, matching the same change made to ms01-01
in the previous session.

## Motivation

Smart plugs depend on AC power cycling and require "Power on after AC loss" BIOS
setting. AMT provides true hardware-level remote power control (power on, reset,
power off) at any time regardless of OS state or Tapo cloud availability. All
three MS-01 machines have i226-LM NICs (port 3) dedicated to AMT, already
configured on the management VLAN.

## Switch Port / VLAN Configuration

The management port (port3) on each MS-01 is already correctly configured:
- UniFi port profile: `management_only`
- Native VLAN: `management` (VLAN 11, 10.0.11.0/24)
- Tagged VLANs: none

AMT expects untagged traffic on its NIC. Because the switch port provides
native VLAN 11 with no additional tagging, AMT at 10.0.11.11 and 10.0.11.12
will receive and respond to untagged frames correctly.

**Note for physical setup:** If AMT was configured in MEBx with a VLAN tag, it
must be cleared (set to 0 / disabled). MEBx path:
`Intel(R) ME Configuration > Intel(R) AMT Configuration > Network Setup >
Intel(R) ME Network Name Settings > VLAN`. Disable or set to 0.
