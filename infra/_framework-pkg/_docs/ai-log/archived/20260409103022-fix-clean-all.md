# Fix framework/clean-all: config reader, UniFi pre-purge, dead code

## Problems fixed

### Broken config readers (silently doing nothing after refactor)
`_load_merged_config` read `providers.<pkg>.config_params` (old format). After the
recent refactor, `config_params` lives at the package top level. So Stage 0 (Proxmox
purge) and Stage 0.5 (GKE purge) were silently finding zero nodes/clusters and
skipping everything.

- Replaced with flat `_all_config_params` aggregation (same approach as `config_base`)
- Added `_ancestor_get(all_params, path, key)` helper for inherited-key lookups
- Added `_load_all_secret_params()` that decrypts all `*.sops.yaml` files once per run
- Stage 0 now scans `_all_config_params` for `_provider_proxmox_endpoint` + `node_name`;
  matches secrets by path last-component (pve-1, pve-2, ms01-01 are unique)
- Stage 0.5 now scans `_all_config_params` for `cluster_name` + `location` +
  `_provider_gcp_project` (via ancestor-merge)
- Also fixed Stage 0 key name: `_provider_endpoint` → `_provider_proxmox_endpoint`
- Also fixed Stage 0 secrets path: old `providers.proxmox.config_params` → new
  `proxmox-pkg_secrets.config_params`

### Missing UniFi pre-purge (resources orphaned after clean-all)
Added Stage 0.4: UniFi VLAN / port-profile purge via API. Runs the same 4-step
teardown sequence as `scripts/ai-only-scripts/destroy-unifi-resources/`:
  1. Clear portconf_id refs from device ports
  2. Clear excluded_networkconf_ids refs from device ports
  3. Delete managed port profiles
  4. Delete managed networks (VLANs)

Finds ctrl_url, vlans, port_profiles from `_all_config_params`; credentials from
`_all_secret_params`. If the controller is unreachable the stage warns and skips —
the ai-only-script remains available as a manual recovery tool.

### Dead code removed
- `STACK_ROOT = GIT_ROOT` alias (unused after cwd arg now uses `_find_infra_dir()`)
- `_find_stack_config()` (used only to populate a variable that was then unused)
- `secrets_config` / `_find_secrets_config()` parameters (replaced by
  `_load_all_secret_params()`)
- `_load_merged_config` no longer merges into `providers.*` structure

## No change to
- Stages 1–5 (unlock, destroy, wipe state, wipe local, move logs)
- `confirm_destructive()` / `--force` / `_FORCE_DELETE=YES`
- `destroy-unifi-resources` ai-only-script (kept as manual recovery tool)
