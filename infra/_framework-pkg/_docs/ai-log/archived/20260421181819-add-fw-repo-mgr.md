## Design

`fw-repo-mgr -b <name>` workflow:
1. Resolve source URL from `source_repos` registry (or inline `source_url`)
2. Clone source repo to `$HOME/git/<name>/` (or pull if already cloned)
3. Prune real directories from `infra/` that are NOT in the target's `framework_packages` list
4. Overwrite `infra/_framework-pkg/_config/framework_packages.yaml` with target package list
5. Run `pkg-mgr --sync` to handle imported packages (symlinks)
6. Commit + optional push to `upstream_url`

The `status` sub-command shows a table of all configured targets with clone status.
