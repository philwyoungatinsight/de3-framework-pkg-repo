# MaaS ms01-01: deployment fixes and lifecycle improvements

## Summary

ms01-01 (MS-01 #1) successfully reached Deployed state (Debian 13/trixie)
after a series of fixes to the MaaS lifecycle scripts and configuration.

## Problems Fixed

### 1. trigger-commission.sh: power_parameters cleared during commission

**Root cause**: Step 4 called `machine update power_type=manual power_parameters={}`,
which actively wiped the AMT credentials stored in MaaS. Step 6 then tried to
restore them, but MaaS rejects power_parameters updates on machines in Commissioning
state (returns 503 or silently ignores).

**Fix**: Removed `power_parameters={}` from step 4 — just set `power_type=manual`.
MaaS preserves power_parameters when only power_type changes. Simplified step 6 to
only restore `power_type` (not parameters, since they were never cleared).

### 2. wait-for-deployed.sh: "unknown" status terminates polling prematurely

**Root cause**: `_check_deployed_done` treated `unknown` (returned by `_status`
when MaaS SSH/API is transiently unreachable) as a terminal state, aborting the
polling loop and failing the provisioner. The machine was still in Deploying state.

**Fix**: Added an explicit `unknown)` case that continues polling with a warning,
matching behavior for other non-terminal states.

### 3. AMT firmware crash on ms01-01

**Symptom**: Power state query times out; MaaS cannot power on machine for deployment.
AMT firmware on ms01-01 crashes after a few minutes of uptime and only recovers
after an AC power cycle.

**Recovery**: Bounce Tapo plug (192.168.1.231) via smart-plug-proxy to restore AC
power, then send `maas machine power-on` once machine is on provisioning VLAN.

### 4. No configurable timeouts for commission/deploy wait

**Fix**: Added `commission_wait_timeout: "1800"` and `deploy_wait_timeout: "3600"`
to ms01-01 machine config in `pwy-home-lab-pkg.yaml`. Wired them into
`commission/ready/terragrunt.hcl` (`wait_timeout`) and
`commission/ready/allocated/deploying/deployed/terragrunt.hcl` (`deploy_timeout`).
Defaults remain 2400s / 10800s for machines that don't set them.

## New Scripts

**`scripts/ai-only-scripts/maas-power-control/`** — Ansible playbook for MaaS
power operations:
- `bounce`: AC power cycle via Tapo smart plug + MaaS power-on
- `power-on` / `power-off`: MaaS API calls
- `status`: print MaaS machine status + live power state

Usage:
```bash
ACTION=bounce SYSTEM_ID=hy8dqm PLUG_HOST=192.168.1.231 PLUG_TYPE=tapo ./run
ACTION=status SYSTEM_ID=hy8dqm ./run
```

## Result

ms01-01 is Deployed running Debian 13 (trixie). Cloud-init ran successfully.
Machine is post-deployment Proxmox node at cloud_public_ip 10.0.10.116.
