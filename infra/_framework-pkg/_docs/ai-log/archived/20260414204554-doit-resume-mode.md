# /doit: Add Resume Mode via Plan Name Arg

## Summary

`/doit` now accepts an existing plan name as its argument to skip straight to execution,
eliminating the need to re-describe the task after clearing context. The detection is
automatic: if `$ARGUMENTS` matches an existing `docs/ai-plans/<arg>.md`, it jumps to
Step 8 (execute); otherwise it proceeds with the normal planning flow.

## Changes

- **`.claude/commands/doit.md`** — added Step 2 (mode detection), renumbered steps 3–8;
  planning steps now marked *(new tasks only)*; Step 7 tells the user to run
  `/doit <kebab-name>` instead of `/clear` then re-describe; Step 8 cross-reference updated

## Notes

The heuristic for "plan name vs task description" is: no spaces = treat as filename stem,
check for `docs/ai-plans/<arg>.md`. If the file doesn't exist, falls through to normal
task-description mode — so a mistyped plan name just starts a new plan rather than erroring.
