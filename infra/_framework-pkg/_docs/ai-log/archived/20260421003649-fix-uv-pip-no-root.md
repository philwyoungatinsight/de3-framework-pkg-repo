# Fix: `uv pip install --system` fails without root

## Root Cause

`uv 0.5.9` dropped support for `--user` as a pip install location flag and requires
`--system` instead when outside a venv. But `--system` targets `/usr/local/lib/python3.10/dist-packages/`
which is root-owned, causing `Permission denied` on `packaging==26.1` install.

## Fix

`install-requirements.sh`: when inside a venv, use uv (fast). When outside a venv,
fall back to `pip install --user` which installs to `~/.local/lib/python3.10/site-packages/`
(where packages like `pyyaml` were already installed). This avoids the root requirement.
