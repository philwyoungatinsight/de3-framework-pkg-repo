# GUI: Persistent Status Bar

**Date**: 2026-04-18  
**Plan**: `docs/ai-plans/gui-status-bar.md`

## What changed

Added a persistent status bar to the bottom of the GUI (`homelab_gui.py`):

- **`show_status_bar: bool = True`** — new state var (persisted to saved config)
- **`toggle_show_status_bar` / `flip_show_status_bar`** — event handlers
- **`app_activity_active`** — computed var; `True` when `app_status_message != ""`
- **`_STATUS_BAR_HEIGHT = "26px"` / `static_status_bar()`** — permanent bottom strip component; shows spinner + message when active, empty when idle; highlighted accent background during activity
- **`index()` updated**:
  - Root `rx.box` now uses `display="flex"`, `flex_direction="column"`, `height="100vh"` to enable proper height distribution
  - `startup_status_banner()` (floating overlay) gated on `~show_status_bar`
  - Main area wrapped in `rx.box(flex="1", min_height="0")` to fill remaining space
  - `static_status_bar()` appended at bottom, gated on `show_status_bar`
  - Three internal `height="100vh"` / `height="calc(100vh - 36px)"` values changed to `height="100%"` so they fill their flex parent rather than the full viewport
- **Appearance → Layout section**: "Status bar" checkbox added above "Floating panels mode"
- Persistence wired in `_save_current_config()` / `_load_saved_menu()`

## Effect

When status bar is enabled (default): a 26px strip always occupies the bottom of the app. It's highlighted (accent colour + spinner) while GCS syncs or inventory refresh runs, empty but present when idle. Terminal output and bottom-row panels are never obscured by the floating banner.

When disabled: falls back to the existing `startup_status_banner()` floating overlay behaviour.
