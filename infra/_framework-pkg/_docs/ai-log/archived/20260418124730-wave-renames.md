# Wave Renames — Clarify Wave Names Across the Stack

## Summary

Six waves were renamed to use a consistent, descriptive naming scheme that reflects
their actual function rather than their historical PXE/provisioning origin. The new
names follow `<domain>.<subdomain>.<action>` conventions. A new doc was created for
the `maas.machine.config.power` wave, and the `infra/de3-gui-pkg/_application/de3-gui/run`
script was restored after being accidentally deleted from disk.

## Changes

- **`config/waves_ordering.yaml`** — six wave renames applied
- **`infra/maas-pkg/_config/maas-pkg.yaml`** — wave names + descriptions updated
- **`infra/proxmox-pkg/_config/proxmox-pkg.yaml`** — `hw.proxmox.storage` renamed
- **`infra/pwy-home-lab-pkg/_config/pwy-home-lab-pkg.yaml`** — all `_wave:` references updated
- **`infra/maas-pkg/_docs/wave-machine-config-power.md`** (new) — documents purpose,
  code, trigger conditions, and recovery for `maas.machine.config.power` wave
- All playbooks, READMEs, docs, and ai-only-scripts updated to use new wave names
- **`infra/de3-gui-pkg/_application/de3-gui/run`** — restored via `git checkout HEAD`
  after being accidentally deleted from disk (caused `./run -A de3-gui` to fail)
