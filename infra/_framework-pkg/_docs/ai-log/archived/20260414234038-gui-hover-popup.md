# GUI: Hover Popup for Node Details

## Summary

Added a floating popup that appears when the cursor rests on a tree node for 5 seconds.
The popup shows two sections: unit-state.yaml data (status, timestamps, exit codes) at the
top, and the full config_params for the node below. It is moveable by dragging the header
and resizable via the CSS resize handle in the bottom-right corner.

## Changes

- **`infra/de3-gui-pkg/_application/de3-gui/homelab_gui/homelab_gui.py`** — all changes:
  - **State vars**: `hover_popup_open`, `hover_popup_node_path`, `hover_pending_path`,
    `hover_popup_unit_state`, `hover_popup_params`
  - **Computed vars**: `hover_popup_unit_state_rows` (dict → ordered list for `rx.foreach`),
    `hover_popup_has_unit_state`, `hover_popup_has_params`
  - **`install_resizer`**: appended `mouse_tracker_js` — installs a `document.mousemove`
    listener that tracks cursor position in `window._mouseX/Y` (used to position the popup)
  - **`start_hover_timer(path)`**: sets `hover_pending_path`; calls JS `setTimeout` 5000 ms
    that stores cursor coords in `window._hoverPopupData` and clicks the hidden trigger div
  - **`cancel_hover_timer()`**: clears `hover_pending_path` (if popup not open); calls JS
    `clearTimeout` to cancel any pending timer
  - **`on_hover_show_trigger()`**: triggered by hidden div click; guards on `hover_pending_path`,
    loads unit-state and params, sets `hover_popup_open = True`, chains to `position_hover_popup`
    via `rx.call_script` callback
  - **`position_hover_popup(data)`**: callback receiving `"x|y"` string; builds and executes JS
    that sets `--popup-x/y` CSS custom properties and wires pointer-capture drag on the header
  - **`close_hover_popup()`**: clears all hover popup state vars
  - **`tree_node_component`**: added `on_mouse_enter=AppState.start_hover_timer(node["path"])`
    and `on_mouse_leave=AppState.cancel_hover_timer` to the row box
  - **`_hover_state_row(item)`**: simple `rx.hstack` component for one unit-state key-value row
  - **`hover_popup_window()`**: full popup component — fixed-position, flex-column, header with
    drag handle + close button, scrollable content area with unit-state and params sections
  - **`index()`**: added `hover_popup_window()` and `rx.el.div(id="hover-show-trigger")` trigger

## Notes

- **Position persists across React re-renders via CSS custom properties**: the popup's `left`/
  `top` style use `var(--popup-x)` / `var(--popup-y)` instead of Reflex state vars. JS drag
  updates both the element's inline style and the CSS vars, so React re-renders never reset
  the drag position.
- **Drag uses pointer-capture**: `hdr.setPointerCapture(e.pointerId)` tracks the pointer even
  when it moves outside the header element at speed, avoiding jittery dragging.
- **Trigger-div pattern** (same as `cy-node-trigger`, `rf-node-trigger`): JS stores data in
  `window._hoverPopupData`, then `.click()`s the hidden div to invoke the Reflex handler.
  This avoids needing to call Reflex event endpoints directly from `setTimeout`.
- **Cancellation guard**: `on_hover_show_trigger` checks `hover_pending_path` before opening —
  if the user left the node before the 5s timer fired (mouse leave clears it), the popup
  does not appear.
- **Resize**: `resize: both; overflow: hidden` on the outer box gives the browser's native
  resize handle; inner content area uses `overflow_y: auto; flex: 1; min_height: 0` so it
  scrolls within the resized container.
