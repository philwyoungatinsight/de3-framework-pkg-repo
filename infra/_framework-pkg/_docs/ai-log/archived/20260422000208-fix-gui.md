## fix-gui — 2026-04-22

Fixed two bugs in `run` that caused `./run -A de3-gui` to fail with
`ERROR: waves_ordering.yaml not found`.

**Bug 1** — `waves_ordering.yaml` discovery used a hand-rolled search that
checked `config/` and `infra/*/_config/` but not the `_framework_settings/`
subdirectory where the file actually lives. Replaced with `find_framework_config_dirs()`
(reversed for highest-priority-first), consistent with how the rest of the script
discovers framework config.

**Bug 2** — `-A` handling was placed after `load_all_configs()` in `main()`,
so the config load (and its now-failing `waves_ordering.yaml` lookup) always ran
even when no wave config was needed. Moved the `-A` early-exit before
`load_all_configs()` and deleted the duplicate block that remained.

Verified: `./run --list-waves` succeeds; `./run -A de3-gui` proceeds into the
application Makefile without hitting config-load errors.
