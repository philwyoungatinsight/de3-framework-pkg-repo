# mgmt_wake_via_plug: smart-plug fallback for unreliable AMT firmware (ms01-01)

## Summary

ms01-01 has broken Intel AMT firmware that crashes after a few minutes of uptime.
AMT is unreachable (port 16993 connection refused) even while the machine is AC-powered
and fully running. Firmware updates and BIOS idle-timeout changes did not fix it.

Implemented `mgmt_wake_via_plug: true` — a new unit_param that causes the auto-import
before_hook to bounce the smart plug when AMT is unreachable, wait for AMT firmware to
reinitialise (~60s), then proceed with the normal CIM boot override + wsman power-cycle.

## Why smart plug + AMT together

- **Smart plug alone** cannot force PXE boot — the machine will boot from its default
  BIOS boot order (NVMe), not from the network.
- **AMT alone** (on ms01-01) is unreliable — firmware crashes after a few minutes.
- **Smart plug → wait for AMT → AMT wsman** gives us both: AC power cycle restores AMT
  standby (~60s after plug-on), then wsman issues the CIM boot override to force PXE.

## Changes

### `infra/maas-pkg/_tg_scripts/maas/auto-import/run`

- Updated `power_cycle_amt()` signature: new params `plug_host=""`, `plug_type="kasa"`,
  `wake_via_plug=False`.
- Added `_amt_reachable(maas_host, amt_ip, timeout_secs=5)`: checks AMT port 16993
  via `nc -z` from the MaaS server.
- Added `_wait_for_amt(maas_host, amt_ip, wake_timeout)`: polls AMT port every 5s
  up to `_MGMT_WAKE_TIMEOUT` seconds (default 120s).
- In `power_cycle_amt()`: if AMT unreachable and `wake_via_plug=True` and `plug_host`
  set → call `power_cycle_smart_plug()` → `_wait_for_amt()` → proceed with wsman.
- Updated `main()`: parses 8th arg `mgmt_wake_via_plug`; passes `plug_host`, `plug_type`,
  `wake_via_plug` to `power_cycle_amt()`.
- Updated module docstring and CLI usage.

### All 4 physical machine `terragrunt.hcl` files (ms01-01, ms01-02, ms01-03, nuc-1)

Added 8th arg to `before_hook "auto_import_maas_machine"` execute list:
```hcl
tostring(try(local.up.mgmt_wake_via_plug, false)),
```
Defaults to `"false"` for all machines without this param set.

### `infra/pwy-home-lab-pkg/_config/pwy-home-lab-pkg.yaml` — ms01-01

- `power_type: manual` → `power_type: amt`
- Added `mgmt_wake_via_plug: true`
- Added `smart_plug_host: ""` (placeholder — set to plug IP once cabled to ms01-01)
- Added `smart_plug_type: kasa`
- Updated `_unit_purpose` to describe the broken-AMT situation

### `infra/maas-pkg/_wave_scripts/test-ansible-playbooks/maas/maas-machines-precheck/playbook.yaml`

- AMT machine list builder now captures `wake_via_plug` field from config.
- Fail task updated: skips machines with `wake_via_plug: true` — these are treated as
  warnings because the auto-import hook handles AMT unreachability itself.

### `infra/maas-pkg/_docs/machine-onboarding.md`

- New `mgmt_wake_via_plug` section in "Fill in machine config" explaining the feature,
  how it works, and the `_MGMT_WAKE_TIMEOUT` env var.
- Added note about ms01-01 broken AMT in the "Run make" section.
- Updated machine table to reflect ms01-01's power type.

## Status

`smart_plug_host` for ms01-01 is currently `""` (empty). The feature is wired up but
the plug bounce will be skipped until the plug IP is set. Once a smart plug is cabled
to ms01-01 and given a static IP (via UniFi `fixed_clients`), fill in:

```yaml
# in pwy-home-lab-pkg.yaml, ms01-01 block:
smart_plug_host: "<plug IP>"
```

Physical interventions still required before `maas.lifecycle.new` passes:
- ms01-01: smart plug cabled, IP set in YAML
- ms01-03: BIOS boot order → PXE first on provisioning NIC
- nuc-1: BIOS power recovery → auto-on; boot order → PXE first
