# Framework Tools

Core automation tools for the de3 stack. These directories are framework internals — not standard package directories.

## Contents

| Directory | Purpose |
|---|---|
| `_utilities/` | Shared bash/python/Ansible library. Key components: `framework-utils.sh` (destructive-action confirmation, `wait_for_condition` polling loop, backend bootstrap, component config lookup), `gcs-status.sh`/`gcs_status.py` (fire-and-forget unit/wave status writes to GCS), Ansible roles, Terragrunt tg-scripts. |
| `_clean_all/` | Nuclear cleanup (`make clean-all`): runs per-package pre-purge scripts, removes all GCS state locks, destroys all waves in reverse order, wipes GCS state bucket, and removes local `.terragrunt-cache`. Accepts `--force` to skip the "NUKE" confirmation prompt. |
| `_ephemeral/` | RAM-backed directory management. `ephemeral` reads `framework.ephemeral_dirs` from `framework.yaml` and mounts each declared directory as a `tmpfs` (Linux) or HFS+ RAM disk (macOS). Idempotent — skipped if already mounted with matching parameters; remounts on size change; applies `chmod`/`chown` on mode/owner change. Skips silently in CI. |
| `_generate-inventory/` | Generates the Ansible inventory (`$_DYNAMIC_DIR/ansible/inventory/hosts.yml`) by reading Terraform remote state from the configured backend (GCS/S3/Azure/local) and cross-referencing per-package YAML config. Derives host groups from `role_*` tags. Called by most tg-scripts and wave-scripts before running Ansible playbooks. |
| `_pkg-mgr/` | Remote package manager. Manages external-repo packages: clones repos into `_ext_packages/`, symlinks packages into `infra/`, detects unit/config collisions. Commands: `sync`, `add-repo`, `remove-repo`, `import`, `remove`, `rename`, `copy`, `status`. Rename/copy handles filesystem, config YAML, SOPS re-encryption, and GCS state migration atomically. |
| `_unit-mgr/` | Terragrunt unit refactor tool. Moves or copies unit trees while keeping filesystem, public config YAML (`config_params`), SOPS secrets, and GCS state blobs in sync. Scans for external `dependencies` references that would break after a move. Integrates with the GUI Refactor panel (right-click → Copy/Move with preview). GCS backend only. |
| `_config-mgr/` | Config pre-processor, reader, and writer. Called from `set_env.sh` via the `generate` entry point (avoids infinite recursion — does not source `set_env.sh`). Commands: `generate` (merge all source YAML into `$_CONFIG_DIR` for fast HCL reads), `get <unit_path>` (print merged `config_params`), `set <unit_path> <key> <value>` (write a `config_params` key to the correct source YAML, optionally `--sops`), `set-raw <pkg> <key_path> <value>` (write an arbitrary key in a package config), `move <src> <dst>` (rename `config_params` keys — called by `unit-mgr`). |
| `_fw-repo-mgr/` | Framework repository manager. Manages target repos that consume the framework: initialises from a source repo (rsync + `git init`, no history copied), prunes `infra/` to the declared package list, writes `framework_packages.yaml` and config scaffolding (`config/_framework.yaml`, minimal `_framework_settings/`), runs `pkg-mgr --sync` to install external packages, then commits and optionally pushes to an upstream remote. Commands: `build [<name>] [--force-push]` (sync all or one named repo), `status` (table of local path, package count, upstream, and git cleanliness). |
| `_git_root/` | Canonical scaffold files for the root of a framework consumer repo: `Makefile`, `root.hcl`, `run`, `set_env.sh`, `CLAUDE.md`, `.sops.yaml`, `.gitlab-ci.yml`, `README.md`. Every repo that uses the de3 framework should have these at its top level. |
| `_fw_repos_diagram_exporter/` | Framework repo discovery and diagram export tool. BFS-discovers all reachable framework repos by cloning them into a dedicated cache (`~/git/fw_repos_diagram_exporter_cache/`) and scanning each for `_framework_settings` directories. Records results in `config/tmp/fw_repos_diagram_exporter/known-fw-repos.yaml`. Renders as yaml, json, text tree, and/or DOT graph simultaneously. Leverages `framework_repo_manager.yaml` to record which repos were generated by which source repos (`created_by` lineage). Configurable auto-refresh with modes `never`, `fixed_time`, `file_age` (default). Config: `_framework_settings/framework_repos_diagram_exporter.yaml`. |
| `_ai-only-scripts/` | AI-generated diagnostic and one-off fix scripts. Each has its own subdirectory with a named entrypoint matching the directory name. Not invoked by automation. |
